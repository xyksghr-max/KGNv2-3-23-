# T6 几何增强混合数据集

Last updated: 2026-04-25

## 目标

T6 第一阶段只验证“数据几何多样性”是否带来正向信号，不继续改网络结构、loss 或推理后处理。

核心对照：

- `primitive-only`：现有 `ps_grasp_single_1k`，800 个训练场景，约 4000 张训练图。
- `primitive + geom-enhanced`：新增 `ps_grasp_single_mix_t6_1k`，总训练规模仍为 800 个训练场景，约 4000 张训练图。

两者使用相同训练预算，例如 `batch_size=1, epoch=5`。第一轮先看 b1/e5 信号，不直接上 b12/e400。

## 实现边界

T6 v1 不使用 Stable Diffusion / ControlNet 纹理增强，也不直接把任意 Text-to-3D `.obj` 当作训练对象。

当前实现采用低风险几何增强：

- 极端比例 `cuboid`：长条、薄板、扁盒。
- 扩展 `cylinder`：更细、更长。
- 扩展 `ring`：更薄、更宽范围。
- 扩展 `stick`：更细、更长。

注意：这些增强对象在 `scene_info.json` 中仍记录为原有 `obj_type`，例如 `cuboid`、`cylinder`、`ring`、`stick`。这样可以复用原来的：

- `generate_grasp_family()`
- 简化夹爪碰撞过滤
- `PSGrasp` loader
- `test.py` 离线评估
- 按类别统计逻辑

因此 T6 v1 是“可控几何分布增强”，不是完整的任意 mesh 自动抓取标注系统。

## 训练命令模板

T6-mixed 训练时使用显式数据目录：

```bash
cd /root/autodl-tmp/KGN-main/src
python -u main.py grasp_pose \
  --exp_id t6_mix_b1_single_r512_e5 \
  --dataset ps_grasp \
  --ps_data_mode single \
  --ps_data_dir ps_grasp_single_mix_t6_1k \
  --batch_size 1 \
  --master_batch_size 1 \
  --lr 1.25e-4 \
  --load_model ../pretrained_weights/ctdet_coco_dla_2x.pth \
  --gpus 0 \
  --num_workers 0 \
  --num_epochs 5 \
  --val_intervals 1 \
  --print_iter 20 \
  --input_res 512 \
  --pnp_type cvIPPE \
  --kpt_type box \
  --input_mod RGBD \
  --sep_scale_branch \
  --scale_weight 10 \
  --no_nms \
  --no_kpts_refine \
  --correct_rl \
  --ori_num 9 \
  --scale_kpts_mode 1 \
  --scale_coeff_k 1 \
  2>&1 | tee ../exp/grasp_pose/t6_mix_b1_single_r512_e5_terminal.log
```

如需验证 T2/T3.4 结构在混合数据上的效果，应在上述命令基础上加入对应开关，但必须与 primitive-only baseline 保持同一组模型参数。

## 成功判据

- T6-mixed 与 primitive-only 使用同样训练图片数、epoch、batch size、学习率和测试阈值。
- 在原 primitive single/multi test 上不明显退化。
- 在几何增强 held-out test 上，T6-mixed 的 `GSR` 或 `GCR` 高于 primitive-only。
- 所有结果必须标明是 b1/e5 快速数据归因实验，不写成最终满预算结论。

已完成 smoke 结果：

- `h5py` 在本地 `kgnv2` 环境可用。
- ACRONYM official repo 已放在 `data/external/acronym_repo/`。
- `ps_grasp_single_mesh_t62_smoke` 已生成：
  - 50 scenes
  - 5 cameras per scene
  - 40 train scenes / 10 test scenes
  - 15000 total grasp labels
  - 6522 non-colliding grasp labels
  - sampled 2D keypoint projection: `3968 / 4000` inside image
- `PSGrasp` loader/training smoke 已跑通：
  - `exp_id=t62_mesh_loader_smoke`
  - 20 iterations
  - no NaN
  - no loader crash

关键修正：

- `get_ori_cls(range_mode=0)` 对 180 度周期的夹爪开口方向做角度 wrapping，避免 mesh 抓取投影产生负 orientation class。
- `PSGrasp.run_eval()` 支持动态新增 `mesh` 类别的 `MetricCalculator`，避免后续 mesh held-out eval 因类别名崩溃。

当前限制：

- 仍只是 ACRONYM sample 级闭环，不代表 full ACRONYM 数据已经接入。
- full ACRONYM 和 ShapeNet mesh 可能需要账号、协议确认或额外下载。
- T6.2 第一阶段还不进入 b1/e5，因为目前 sample 只有单一 Mug，无法形成有意义的训练对照。

当前结论：

- 我们已经具备“数据资产审计工具”和 sample 级回归能力。
- full ACRONYM `.h5` 标签规模已经足够，能够支持“多类别、多物体、多成功抓取姿态”的后续数据构建。
- 现在的硬阻塞是 ShapeNetSem mesh：ACRONYM 官方 README 要求另行从 ShapeNet 下载 ShapeNetSem meshes；当前 ShapeNet 下载入口需要登录。
- 在 mesh 未就绪前，不能进入 `ps_grasp_single_mesh_t62_1k` 或 b1/e5，因为无法渲染 RGB-D，也无法完成 mesh collision/filtering。
- 下一步需要用户注册/登录 ShapeNet 并下载 ShapeNetSem mesh，或提供已有本地 ShapeNetSem mesh 路径。建议放置为 `data/external/acronym/meshes/<Category>/<object_id>.obj`，并继续使用 `--mesh_root ../data/external/acronym`。
