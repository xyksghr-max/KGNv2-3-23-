# T6.3 GraspNet-to-PS Feasibility Audit

Last updated: 2026-04-27

## Purpose

T6.3 is a dataset-side pilot. It does not modify the KGN network, loss, decoder, PnP recovery, or post-processing chain.

The goal is to test whether GraspNet object-level mesh models and 6D grasp labels can become a new PS-style training data source for KGN-main:

```text
GraspNet object mesh + grasp_label.npz
-> synthetic RGB-D render
-> PS-style scene_info.json
-> 2D keypoint labels
-> PSGrasp loader smoke
```

## GraspNet88 Single-Object Eval Close-Out

The follow-up all-object synthetic mesh evaluation set is complete:

```text
data/ps_grasp_single_graspnet_t63_eval_88obj
```

Dataset properties:

- 88 GraspNet object meshes, object ids `000..087`
- 5 rendered RGB-D views per object
- 440 evaluation samples
- `train.txt` is empty
- `test.txt` covers all 88 scenes
- labels are GraspNet object-level grasps converted to the current KGN/PS-style `4x4 pose + width` format
- this is a synthetic single-object mesh evaluation set, not a real-camera single-object GraspNet split

KGNv2 + Refine-Scale evaluation:

| threshold | GSR / GCR / OSR | image success | object any-view success |
| --- | --- | ---: | ---: |
| `1cm + 20deg` | `0.0614 / 0.0135 / 0.3750` | `165 / 440` | `67 / 88` |
| `2cm + 30deg` | `0.2868 / 0.0864 / 0.8136` | `358 / 440` | `86 / 88` |
| `3cm + 45deg` | `0.5424 / 0.2696 / 0.9364` | `412 / 440` | `87 / 88` |

Interpretation:

- `KGNv2 no-refine` is the closest original-style paper-2/KGNv2 external-evaluation protocol.
- `KGNv2 + Refine-Scale` is a project-level inference-side RGB-D scale refinement improvement.
- Under the relaxed Refine-Scale protocol, the primitive-trained official KGNv2 checkpoint covers `87 / 88` complex GraspNet mesh objects.
- The only relaxed no-success object is `065`, which has a high-aspect-ratio geometry and only `2` non-colliding GT grasps in the converted set.
- Small objects are not the main failure source in this synthetic single-object setup; larger, elongated, high-aspect-ratio, or sparse-label objects are harder.

## Next Step

The GraspNet88 synthetic single-object mesh evaluation itself is closed. Future work should use these results for thesis tables, qualitative figures, and failure analysis rather than expanding the dataset again unless a new controlled question is defined.
