# 云服务器实验命令记录

## 1. 文档用途

本文档记录 `KGN-main` 在 AutoDL 云服务器上的环境配置、依赖安装、DCNv2 编译、项目级运行检查、数据准备、远程开发与 Codex 接入、云端训练评估与实验结果回传相关命令。

历史环境配置阶段曾只负责：

- 云服务器环境搭建
- 依赖安装
- NMS 编译
- DCNv2 编译
- KGN-main 最小可运行检查
- 训练前数据与权重准备

历史环境配置阶段曾不负责：

- 论文学习
- 算法改造
- 实验结论判断
- 正式长训练
- 直接修改 KGN-main 算法代码

阶段更新：

- 截至 2026-04-16，云服务器环境已完成可训练、可测试最小闭环验证。
- 截至 2026-04-18，云端已同步 `feat/t3-prob-pose-loss-clean`，并确认 `T3.1` 关键文件和 `KGN-Pro-main` 参考目录存在。
- 截至 2026-04-19，主工作流调整为“本地开发 + GitHub 中转 + 云端训练/评估 + 单实验目录回传”。
- 当前云服务器主要角色是训练/评估执行端：`pull` 最新分支、跑 GPU smoke train、正式训练、测试评估、保存日志和实验结果。
- 远程 Codex 已完成配置并可作为备用工具，但由于网络链路和沙箱限制，不再作为主要代码修改入口。

重要约束：

- 云端主路径固定为 `/root/autodl-tmp/KGN-main`
- 不使用 `/home/xyk/KGN-main` 作为云端主路径
- 不把项目、环境缓存、大数据放系统盘
- 不在 `kgnv2` 主环境安装 `pyro-ppl`
- 不整体照搬 `KGN-Pro-main` 的 requirements
- 不为 `mayavi` 卡住环境主线
- RTX 4090 + torch 1.13.1 编译扩展时不使用 `TORCH_CUDA_ARCH_LIST=8.9`
- DCNv2 优先使用系统 CUDA 11.8
- 正式长训练最后必须用 `tmux` 或 `screen`
- 远程 Codex 沙箱在 AutoDL 容器内不可用，如需临时使用远程 Codex，只能依赖项目规则、Git diff 和人工确认约束高风险操作
- 云端实验结果回传时只压缩单个 `exp/grasp_pose/<exp_id>` 目录，不压缩整个 `exp/grasp_pose/`

## 10. KGN 模型级前向检查

第一次模型检查脚本误传 `opt=opt`：

```text
TypeError: create_model() got an unexpected keyword argument 'opt'
```

原因：

当前仓库 `create_model` 签名为：

```python
create_model(arch, heads, head_conv, inp_channel=3)
```

修正后的检查命令：

```bash
cd /root/autodl-tmp/KGN-main/src

python - <<'PY'
import _init_paths
import torch

from opts import opts
from models.model import create_model

opt = opts().init([
    'grasp_pose',
    '--dataset', 'ps_grasp',
    '--input_mod', 'RGBD',
    '--arch', 'dla_34',
    '--kpt_type', 'box',
    '--pnp_type', 'cvIPPE',
    '--sep_scale_branch',
    '--conf_branch',
    '--no_nms',
    '--no_kpts_refine',
    '--ori_num', '9',
    '--scale_kpts_mode', '1',
    '--scale_coeff_k', '1',
    '--gpus', '0'
])

model = create_model(opt.arch, opt.heads, opt.head_conv, opt.inp_channel)
model = model.cuda()
model.eval()

x = torch.randn(1, 4, 512, 512, device='cuda')
with torch.no_grad():
    y = model(x)

print("model type:", type(model).__name__)
print("output type:", type(y))
if isinstance(y, list):
    print("num outputs:", len(y))
    print("output keys:", sorted(y[0].keys()))
else:
    print("output keys:", sorted(y.keys()))
print("KGN model forward ok")
PY
```

已确认结果：

```text
heads {'hm': 9, 'w': 9, 'kpts_center_offset': 72, 'reg': 2, 'scales': 9, 'conf': 9}
model type: DLASeg
output type: list
num outputs: 1
output keys: ['conf', 'hm', 'kpts_center_offset', 'reg', 'scales', 'w']
KGN model forward ok
```

说明：

- `opts.py` 正常。
- T2 分支 `--conf_branch` 生效。
- DLA + DCNv2 模型构建成功。
- RGBD 4 通道输入可用。
