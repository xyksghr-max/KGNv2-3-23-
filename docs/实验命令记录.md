# 实验命令记录

本文档用于记录 `KGN-main` 项目中已经使用过、后续可能继续复用的训练、测试与对照实验命令。  
它的作用不是替代 `已完成工作.md` 或 `新阶段继续改进计划.md`，而是作为“可直接复制的命令索引”，供新对话、云服务器环境和后续实验复现使用。

当前整理时间：`2026-04-18`

## 1. 使用原则

- 本地默认工作区：`~/KGN-main`
- AutoDL 云端推荐工作区：`/root/autodl-tmp/KGN-main`
- 如果在 AutoDL 云端执行命令，需要把命令开头的 `cd ~/KGN-main/src` 改为：

```bash
cd /root/autodl-tmp/KGN-main/src
```

- 长时间训练建议使用 `tmux` 或 `screen`，不要直接依赖 VSCode Remote-SSH 终端。
- 不同训练预算的实验结果不能直接混在同一个主对照表中比较。
- `T3.1` 当前是训练侧概率姿态辅助损失原型，已经完成代码接入，但正式验证尚未完成，不能写成“已验证有效”。
- 当前 `T3.1` 基线分支为 `feat/t3-prob-pose-loss-clean`，已推送到 GitHub 并同步到云服务器。
- 后续云端验证或修复不应直接堆在该基线分支上，应从当前最新确认分支新建功能/验证分支。

## 3. T2 几何感知置信度增强训练命令

实验含义：

- 训练 `T2` 几何感知置信度增强模型。
- 训练预算为本地小显存友好版本：`batch_size=1`，`num_epochs=5`。
- 使用 `conf_branch` 与 `conf_weight=0.1`。

```bash
cd ~/KGN-main/src
python -u main.py grasp_pose \
  --exp_id t21_geom_conf_single_r512_e5_800200 \
  --dataset ps_grasp \
  --ps_data_mode single \
  --batch_size 1 \
  --master_batch_size 1 \
  --lr 1.25e-4 \
  --load_model ../pretrained_weights/ctdet_coco_dla_2x.pth \
  --gpus 0 \
  --num_workers 0 \
  --num_epochs 5 \
  --val_intervals 1 \
  --print_iter 20 \
  --input_res 512 \
  --pnp_type cvIPPE \
  --kpt_type box \
  --input_mod RGBD \
  --sep_scale_branch \
  --scale_weight 10 \
  --conf_branch \
  --conf_weight 0.1 \
  --no_nms \
  --no_kpts_refine \
  --correct_rl \
  --ori_num 9 \
  --scale_kpts_mode 1 \
  --scale_coeff_k 1
```

## 5. T3.1 概率姿态辅助损失稳定性测试命令

实验含义：

- 用于 `T3.1` 的小规模稳定性验证。
- 只跑 `1` 个 epoch、`200` 个 iteration。
- 目标是检查：
  - 训练是否能正常启动；
  - `prob_pose_loss` 是否进入训练日志；

  - 关闭/开启新 loss 的行为是否可控。
- 该命令不能证明 `T3.1` 已经有效，只能证明原型具备继续验证的基础。

## 7. 原 KGNv2 测试脚本阈值参考

原测试脚本 `experiments/test_kgnv2.sh` 会依次评估三组阈值：

```text
Distance_Th = 0.01 m, Angle_Th = 20°
Distance_Th = 0.02 m, Angle_Th = 30°
Distance_Th = 0.03 m, Angle_Th = 45°
```

之前用于中期答辩展示的原始 KGNv2 `multi -> single` 测试结果为：

```text
Distance_Th = 0.01 m, Angle_Th = 20°:
GSR / GCR / OSR = 83.72% / 60.84% / 99.70%

Distance_Th = 0.02 m, Angle_Th = 30°:
GSR / GCR / OSR = 92.45% / 72.54% / 99.90%

Distance_Th = 0.03 m, Angle_Th = 45°:
GSR / GCR / OSR = 95.50% / 80.00% / 100.00%
```

其中：

- `GSR`：Grasp Success Rate，抓取成功率；
- `GCR`：Grasp Coverage Rate，抓取覆盖率；
- `OSR`：Object Success Rate，物体成功率。
