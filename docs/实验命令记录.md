# 实验命令记录

本文档用于记录 `KGN-main` 项目中已经使用过、后续可能继续复用的训练、测试与对照实验命令。  
它的作用不是替代 `已完成工作.md` 或 `新阶段继续改进计划.md`，而是作为“可直接复制的命令索引”，供新对话、云服务器环境和后续实验复现使用。

当前整理时间：`2026-04-24`

## 1. 使用原则

- 本地默认工作区：`~/KGN-main`
- AutoDL 云端推荐工作区：`/root/autodl-tmp/KGN-main`
- 当前主工作流：本地负责代码修改、文档更新、轻量检查、commit 和 push；云端负责 pull 最新分支、GPU 训练、测试评估和保存实验结果。
- 如果在 AutoDL 云端执行命令，需要把命令开头的 `cd ~/KGN-main/src` 改为：

```bash
cd /root/autodl-tmp/KGN-main/src
```

- 长时间训练建议使用 `tmux` 或 `screen`，不要直接依赖 VSCode Remote-SSH 终端。
- AutoDL 当前镜像未必预装 `tmux`；如果 `tmux: command not found`，使用 `screen`。
- 不同训练预算的实验结果不能直接混在同一个主对照表中比较。
- `b1/e5` 命令只作为快速归因命令，不作为最终完整训练命令。
- 云端实验结束后，不压缩整个 `exp/grasp_pose/`，只压缩本次 `exp/grasp_pose/<exp_id>` 单实验目录，再通过 FileZilla 下载回本地。
- 本地收到云端实验压缩包后，解压到 `/home/xyk/KGN-main/exp/grasp_pose/`，再由本地 Codex 读取日志、analysis 和可视化文件继续分析。
- `T3.1` 当前是训练侧概率姿态辅助损失原型，已经完成代码接入和短预算 control/prob 消融；目前只能写作相对 `ctrl_t2off` 有恢复性正向信号，不能写成“已验证有效”。
- 当前 `T3.1` 代码基线分支为 `feat/t3-prob-pose-loss-clean`，但云端当前运行分支是 `diag-t2-d4ff8ca-base-ablation @ d4ff8ca`。
- 后续云端验证或修复不应直接堆在该基线分支上，应从当前最新确认分支新建功能/验证分支。

## 7. 原 KGNv2 测试脚本阈值参考

原测试脚本 `experiments/test_kgnv2.sh` 会依次评估三组阈值：

```text
Distance_Th = 0.01 m, Angle_Th = 20°
Distance_Th = 0.02 m, Angle_Th = 30°
Distance_Th = 0.03 m, Angle_Th = 45°
```

之前用于中期答辩展示的原始 KGNv2 `multi -> single` 测试结果为：

```text
Distance_Th = 0.01 m, Angle_Th = 20°:
GSR / GCR / OSR = 83.72% / 60.84% / 99.70%

Distance_Th = 0.02 m, Angle_Th = 30°:
GSR / GCR / OSR = 92.45% / 72.54% / 99.90%

Distance_Th = 0.03 m, Angle_Th = 45°:
GSR / GCR / OSR = 95.50% / 80.00% / 100.00%
```

其中：

- `GSR`：Grasp Success Rate，抓取成功率；
- `GCR`：Grasp Coverage Rate，抓取覆盖率；
- `OSR`：Object Success Rate，物体成功率。

## Primitive single per-class GSR statistic fix

- 在 primitive single-object eval 中发现单独 6 类的 GSR 数值异常偏低。

- file: `src/lib/datasets/dataset/ps_grasp.py`

```text

```

```text

```
