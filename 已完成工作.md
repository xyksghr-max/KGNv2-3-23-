### 5.3 P4 v1（第一版训练侧位姿正则）实验记录

#### 5.3.1 实现内容

新增参数：

新增日志项：

- `pose_proxy_mean（代理位姿误差均值）`
- `pose_weight_mean（概率式权重均值）`
#### 5.3.2 指标含义
- `pose_reg_loss`
  - 训练阶段新增的 `pose proxy regularizer（代理位姿正则损失）`
  - 目的：在不改当前确定性推理链的前提下，给几何相关分支增加更强训练约束
- `pose_proxy_mean`
  - 没乘权重之前的原始代理位姿误差均值
  - 用于判断“真正的代理位姿误差是否在下降”
- `pose_weight_mean`
  - 由 `confidence（置信度）` 派生出来的平均权重大小
  - 用于判断 `probabilistic weighting（概率式加权）` 的作用强弱
#### 5.3.3 `pose_reg_loss` 优化谁
- `pose_reg_loss` 直接优化：
  - `kpts_center_offset（关键点中心偏移头）`
  - `scales（尺度头）`
  - `reg（中心偏移头）`
- 也会间接优化：
  - 共享 `backbone（主干网络）`
- 不会直接通过该损失优化：
  - `conf（置信度头）`
原因：
- 当前实现使用了 `conf_quality.detach()`
- 即用 `confidence（置信度）` 做权重，但截断了该分支的梯度

5 轮训练后的验证损失趋势：

#### 6.9.1 这次改了什么

核心变化：

- 保持 `conf_fusion（置信度融合）` 推理使用方式不变；
- 将 `ConfLoss（置信度损失）` 的监督目标，从原来的：
  - detached 的 `kpts_center_offset` 误差
  升级为：
  - 在 `output-resolution（输出分辨率）` 平面中重构后的 `2D keypoint geometry error（二位关键点几何误差）`

当前新版本 `ConfLoss（置信度损失）` 的语义是：
- `conf（置信度）` 不再只是一个更偏“排序分数”的轻量辅助量；
- 它开始更接近论文3里：
  - `confidence map（置信图）`
  - `weighted correspondence（加权对应关系）`
  的低风险近似版本。

#### 6.9.2 怎么实现的
当前实现逻辑位于：
- [src/lib/trains/grasp_pose.py](/home/xyk/KGN-main/src/lib/trains/grasp_pose.py#L19)

具体做法是：
1. 在 `GT center（真值中心）` 的 `ind（索引） + ori_cls（方向类别）` 上，先取出：
   - `reg_pred / reg_gt`
   - `kpts_center_offset_pred / kpts_center_offset_gt`
2. 在 `output-resolution（输出分辨率）` 坐标系下重构：
   - `center_pred（预测中心）`
   - `center_gt（真值中心）`
   - `kpt_pred（预测关键点）`
   - `kpt_gt（真值关键点）`
3. 计算每个抓取的：
   - `geom_err（几何代理误差） = mean(abs(kpt_pred - kpt_gt))`
4. 使用 detached 的 `geom_err（几何代理误差）` 去监督 `conf（置信度）`
5. 仍然采用异方差形式：
   - `loss_conf = exp(-u) * geom_err + u`
   - 其中 `u` 是预测出来的 `log-uncertainty（对数不确定性）`

同时，本次新增了训练统计项：
- `conf_geom_proxy_mean（几何代理误差均值）`

它用于观察当前 batch 中，被 `conf（置信度）` 学习的“几何可靠性信号”大小。

#### 6.9.6 当前评估口径的一个重要注意事项
当前需要明确区分：

- `The metrics for each shape（按类别打印指标）`

当前代码中，按类别打印的 `GSR（抓取成功率）` 存在已知统计口径问题，原因在：
- [src/lib/datasets/dataset/ps_grasp.py](/home/xyk/KGN-main/src/lib/datasets/dataset/ps_grasp.py#L379)
- [src/lib/datasets/dataset/ps_grasp.py](/home/xyk/KGN-main/src/lib/datasets/dataset/ps_grasp.py#L406)
- [src/lib/utils/metric.py](/home/xyk/KGN-main/src/lib/utils/metric.py#L14)

具体来说：
- 每张图像的 `N_grasps（预测抓取数）` 先被加到了所有 shape 的 `grasp_pred_num（预测抓取总数）` 上；
- 但后面逐 object 更新时，又调用的是：
  - `update(0, gt_num, pred_succ_num, gt_cover_num)`
- 因而类别级 `GSR（抓取成功率）` 的分母口径被污染。

所以当前可以这样使用指标：
- `The metrics for all（整体指标）` 中的 `GSR / GCR / OSR`：可以作为主结论使用；
- `shape_summary.json（按类别汇总文件）` 中的：
  - `images_with_any_eval_success`
  - `eval_successful_prediction_total`
  - `accepted_reprojection_error_stats`
  仍然有参考价值；
- 但“按类别单独打印出来的 `GSR（抓取成功率）`”当前不应直接写进论文作为可信结论。
